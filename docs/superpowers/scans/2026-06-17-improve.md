## Patterns observed by lenses

**Architect:** Uneven maturity across three subsystems — `extract/` (oldest, PDF-only) predates the design decision now baked into `outlook/` and `fees/` (both PDF+text). Validate-retry-once is core control flow, not opinionated per subsystem; should live in `extract/client.py`. Subsystem boundaries are otherwise clean — no cross-subsystem calls in implementations.

**Control inversion:** Extract functions correctly accept `client` as parameter (good), but CLI commands build their own client via `_build_client()` (bad). Classic "right pattern at one layer, wrong pattern at another." Pure compute pipeline (`assemble`, `fee_breakdown`) is clean — no client dependencies.

**Test skeptic:** Suite is clean and honest. No flaky tests, no sleeps, no real network. Three subsystems have mirrored test structure (FakeClient, schema round-trip, fixture contracts). Missing: computed-fee-to-fixture parity test (regression in `fee_breakdown()` would diverge from committed fixture silently).

**Dead code:** Codebase has excellent comment hygiene. Only one orphan (`categorize()`); kept honest by tests, just unused by production. Symmetric module structure across the three subsystems.

**Types:** Enum maturity mismatch — `StrataExtract` has three proper enums (`DocType`, `Confidence`, `ReserveTrend`), but `Meeting.type` and `TimelineEvent.type` dodge the same pattern with bare strings. Tool factories and the client's extraction seam all return bare `dict` — three sites where shape is known only to readers.

**Observability:** Codebase trusts that if Claude returns any content, it will deserialize correctly. Doesn't observe what Claude actually returns when surprised — the `$ref` risk flagged in project memory is currently undebuggable.
